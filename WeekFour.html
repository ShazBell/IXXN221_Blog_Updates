User Flow

In today's tutorial, we were tasked with creating a user flow diagram to help Spongebob visualise and achieve his gym goals. In the user flow diagram, we had to think about how he would interact with and navigate a new website that recommends the right meals and workouts.

I collaborated with Adam on this task. This was a task that I personally found difficult to think of from a design standpoint because I couldn't put myself in the mindset of a user going through a new website and each individual step because I don't often think of such things when using a website as I just follow the route that appears most familiar but also because I rarely use any more than the four main websites and Blackboard.

When we shared our user journeys, we discovered that we both had blindspots. We used two different user flow methods, one focusing on finding a website and navigating to create an account, and the other on the side of recommended meals and workout plans. However, I believe that combining the two user flows would provide a sufficient user flow to achieve Spongebob's.

As a designer, this taught me that collaboration is an essential tool in understanding a given brief because the use of various perspectives and approaches of others notice different aspects and bringing ideas together will give a better overall result.

 

 
 

 

 

 

User Journey Map

Following this task, we had to create a customer journey map that analyses what the customer is thinking on the journey to reaching their goal on the website; this was slightly easier because we worked in pairs to create it, but it was still difficult to work on discovering the customer's expectations of the website or what could be improved in the customer journey map.

Understanding the common pain points of the customer and optimising the customer experience is something I found interesting and could be influenced by as a designer, similar to what we looked into for Interaction last year.

 

 
 

 

 

 

 

CSS Box Model

Today's coding assignment was to use CSS Box Model to create a box with margins, borders, padding, and content. I don't believe I completed the task exactly as expected, but I did enjoy playing around with CSS and seeing how different elements changed the style of the elements on the page and what I could do with them. As a designer, this is a critical point of experimentation that can be applied to web design because trying new things means creating something cool and unique on your website that attracts traffic.  

 

Throughout this week at home, I began to create my customer journey map and user flow diagram with a better understanding of their purpose and use in creating a website, putting the user first and allowing the design to influence the user rather than the other way around.
