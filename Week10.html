Editor X Fundamentals

 

I decided for the second part of the assignment that I would do it in CMS as it seemed like it would be pretty straight forward transitioning it from the Figma version to the Editor X format, also as I believe my design is a fairly good standard to build upon whereas my coding abilities are incompatible to even begin to think of how my website would be created in code with the hover and slider effects.

I wanted to focus on the main laws that were discussed extensively in the previous two projects in this project, so I tried to keep the function, layout, and aesthetics of the website as close to my Figma prototype as possible, with a few minor re-considerations due to the different flow of EditorX to Figma.

As we did not have class this week it gave me some time to focus my energy towards the amount of work that needed to be executed for the second part of this assignment. In the pre-recorded lecture we were introduced to the fundamentals of Editor X which seemed simple enough so I began to work on creating the title page and very quickly realised to make my project was going to be a lot more work to recreate in Editor X than I thought it was going to be as it did not have the same functionality as Figma allowing me to have the sort of transitions that made my website feel like an experience rather than just another HTML webpage.

I also discovered that the desktop version was set for an iMac rather than the size I had prototyped with, which was a MacBook screen, which meant that scaling and moving my Figma design selections over to Editor X was a challenge. (This is written in the transcript for 2.2.) At this point, I had grown distant from Editor X because, after spending hours attempting to work on a single page, I was getting nowhere with it looping around and around in circles with no further progress.
