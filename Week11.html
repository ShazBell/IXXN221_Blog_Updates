Thinking like a coder - Assignment 2.2 Development

 

I returned to Editor X over the weekend with a mission to get my website to where it needs to be. I started by revisiting the title page and getting it right, as well as crafting the log-in and sign-in pages. However, because it wasn't very responsive, I created a laptop version to match it, but when I arrived on Monday, everything on my desktop version was gone, and I had some strange issues like the text on the side of the page and the page not taking up the full height, which was just a pain.

However, while browsing the various tools and options in EditorX, I discovered the option to create a dynamic page. I went with the blank/customisable page option so that I could build it in the same way that I do with my feature and work pages in Figma. This tool appears to be very useful for the type of website I'm building, as I was able to make the galleries swipeable and have the "feature" art at the top of the page. When you use this, the image will take you to the appropriate page. This appeared to be a great idea until it wasn't.

I spent a lot of time this week trying to get the dynamic pages to work for all of the different categories, but when I changed something on one page, it changed everything else. I tried researching why this was happening, but there was almost nothing about it online, which wasn't much help, so I couldn't find a solution.

This week at university, I spent all of my time working on creating all of the other pages that I could, such as the profile and gallery and finding solutions in EditorX for transitions, but it did feel more difficult than it needed to be, such as lightboxes to enlarge and show an image in full when clicked on in my gallery.

However, this taught me as a designer to be adaptable to change and different variations of creating the same product, which, as demonstrated by this part of the assignment, is critical to web design. When testing the live versions of the site, I discovered that an iMac or any device that requires a mouse cannot horizontally scroll on the dynamic pages with the current configuration; however, with accessibility as my top priority, I found a way to allow a scroll bar for desktop/laptop sizes.

I was hoping that after my last hand-in, I would be able to get some critical feedback that I could work on to improve the outcome of this website, but because I received very little critical feedback, I couldn't adapt any further to the website at this point and continued to build it in the same manner as in Figma. I've become more accustomed to Editor X as time has passed, but I'm still not a fan of it.
