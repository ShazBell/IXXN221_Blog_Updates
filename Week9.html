Assignment 2.1 Presentations

 

In class, we presented our high-fidelity prototype as well as an overview of our user, the UX Laws, and the process of getting to our high-fidelity prototype. It was discussed in my feedback that features such as the hamburger drop-down seemed out of place with the design and amount of content on the page.

And the addition of things like the arrows pointing down was not a favourite because they did not match the aesthetic of the rest of the webpage, but I had just used these as methods of navigation for the user if they did not know to scroll down for whatever reason. In addition, I believe that the addition of elements such as preferences and log-in error pages would have allowed for a more comprehensive prototype. Additionally, the project page could be pushed further to engage the customer.

What I didn't realise in this project was that we were supposed to create a mobile version of our high fidelity prototype but I had read the added note which turns out was for 2.2, not 2.1 of only needed to be functional for desktop so after my classes I had to sit down and make my final desktop version responsive for mobile.

I knew it wouldn't be too difficult because I had built the website from a mobile iteration, but I had to think about certain elements in different ways, such as how the hover effect on the content page would not work on mobile because a hover cannot be detected, so I had to consider how such pages would work for users in a mobile format.
